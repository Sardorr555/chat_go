Getting Started with Swipies AI Platform

Our platform focuses on making advanced AI technology accessible with minimal technical overhead.

Quick Start Guide

Website Integration

Integrate your LLM-powered chat agent into any website with a simple script tag. The Swipies AI Platform provides an easy-to-use widget that can be customized to match your website's design.

Website Integration: Step-by-Step Guide

Follow these steps to add your AI chatbot to any website:

1
Create Your Bot

Go to the My Bots page and click "Create New Bot". Give your bot a name and description that matches its purpose on your website.

Tip: Upload files or website URLs that contain information your bot should know about your business, products, or services.

2
Get Your Integration Code

After creating your bot:

  1. Click on your bot in the My Bots list
  2. Go to the Integrations tab
  3. Find the Website Integration section
  4. Click "Copy Script" to copy your unique integration code

Important: Each bot has its own unique script tag with a specific ID. Make sure you're using the correct one for the bot you want on your website.

3
Add the Script to Your Website

Open your website's HTML file and paste the script tag into the <head> section:

<!DOCTYPE html>
<html>
<head>
    <title>Your Website</title>
    
    <!-- Other head elements like meta tags, CSS -->
    
    <!-- Swipies AI Chatbot Integration - Paste this script tag -->
    

        ⚠️ WARNING: This chatbot integration is for TEST ENVIRONMENTS ONLY. Do not use in production deployments as it may cause stability issues or data leakage. Use a proper API key for production.
    

    <script src="https://swipies.ai/agent.js?id=YOUR_AGENT_ID" async></script>
</head>
<body>
    <!-- Your website content -->
</body>
</html>

That's it! After adding the script tag, the chat widget will automatically appear on your website. No additional configuration is required.

Working with CMS Platforms

If you're using a CMS like WordPress, Shopify, or Wix:

  • WordPress: Add the script in Appearance → Theme Editor → Theme Header or use a header/footer plugin
  • Shopify: Go to Online Store → Themes → Edit HTML/CSS → Layout → theme.liquid
  • Wix: Go to Settings → Advanced → Custom Code → Add Custom Code → Header
Testing Tip

You can test your integration locally before deploying it to your production website. The chat widget will appear in the bottom-right corner of your website by default.

Telegram Bot Integration

Create a Telegram bot powered by your Swipies AI agents. This allows you to engage with users through the Telegram messaging platform, which has over 500 million active users worldwide.

Why Telegram Bots?

Telegram bots offer a powerful way to reach users where they already are. They can provide customer support, deliver content, answer questions, or even facilitate transactions—all powered by your Swipies AI agents.

Bot Setup

Follow these steps to create and connect your Telegram bot:

1. Find Your Swipies API Key

First, you need your API key to connect your Telegram bot to your Swipies AI agent:

  1. Go to the My Bots page in your dashboard
  2. Find your bot in the list and click on it
  3. Go to the Integrations tab
  4. Look for the API Key section and copy your unique API key
  5. Save this API key securely—you'll need it for your Telegram bot
Finding Your API Key

Your API key is unique to each bot you create. Make sure you're using the correct API key for the bot you want to connect to Telegram.

2. Create a Bot with BotFather

Next, create your Telegram bot using BotFather, Telegram's official bot creation tool:

  1. Open Telegram and search for "@BotFather"
  2. Start a chat and send the command /newbot
  3. Follow the prompts to name your bot and create a username (must end in "bot")
  4. BotFather will provide a token—save this token securely
Security Warning

Keep your Telegram bot token and Swipies API key confidential. Anyone with these tokens can control your bot and access your Swipies account. Never commit them to public repositories or share them in insecure channels.

3. Using Python with Aiogram

If you prefer using Python instead of Node.js, you can create your Telegram bot using the Aiogram framework:

# Install the required packages
pip install aiogram python-dotenv requests

Create a new Python file named bot.py with the following code:

import os
import logging
import requests
from dotenv import load_dotenv
from aiogram import Bot, Dispatcher, executor, types

# Load environment variables
load_dotenv()

# Configure logging
logging.basicConfig(level=logging.INFO)

# Initialize bot and dispatcher
TELEGRAM_BOT_TOKEN = os.getenv("TELEGRAM_BOT_TOKEN")
SWIPIES_API_KEY = os.getenv("SWIPIES_API_KEY")
SWIPIES_API_URL = "https://api.swipies.ai/chat"  # Adjust as needed

bot = Bot(token=TELEGRAM_BOT_TOKEN)
dp = Dispatcher(bot)

# Handler for /start command
@dp.message_handler(commands=["start"])
async def send_welcome(message: types.Message):
    await message.reply("Hi! I'm your AI assistant powered by Swipies. Ask me anything!")

Add the message handler that will connect to your Swipies AI:

# Handler for all text messages
@dp.message_handler()
async def handle_all_messages(message: types.Message):
    user_id = message.from_user.id
    user_text = message.text
    
    # Show typing status while waiting for AI response
    await bot.send_chat_action(message.chat.id, "typing")
    
    try:
        # Call the Swipies AI API
        response = requests.post(
            SWIPIES_API_URL,
            json={
                "message": user_text,
                "apiKey": SWIPIES_API_KEY,
                "userId": str(user_id)  # Optional for tracking conversations
            },
            timeout=30  # Set a reasonable timeout
        )
        
        if response.status_code == 200:
            ai_response = response.json().get("answer")
            await message.reply(ai_response)
        else:
            await message.reply(f"Sorry, I encountered an error: {response.status_code}")
    
    except Exception as e:
        await message.reply(f"Sorry, something went wrong: {str(e)}")

# Main execution point
if __name__ == "__main__":
    print("Starting Swipies AI Telegram bot...")
    executor.start_polling(dp, skip_updates=True)

Here's a simple example of how to use the Swipies AI API in Python:

import requests
import os
from dotenv import load_dotenv

# Load API key from .env file (recommended for security)
load_dotenv()
API_KEY = os.getenv("SWIPIES_API_KEY")
API_URL = "https://api.swipies.ai"

# Send a query to the AI agent
def ask_question(question, agent_id=None):
    payload = {
        "message": question,
        "apiKey": API_KEY
    }
    
    # Optionally specify a particular agent
    if agent_id:
        payload["agentId"] = agent_id
    
    response = requests.post(
        f"{API_URL}/chat",
        json=payload
    )
    
    if response.status_code == 200:
        return response.json()["answer"]
    else:
        return f"Error: {response.status_code} - {response.text}"

# Example usage
question = "What are the key benefits of using Swipies AI Platform?"
answer = ask_question(question)
print(f"Q: {question}\nA: {answer}")

Working with Files and Data Sources

import requests
import os
from dotenv import load_dotenv

load_dotenv()
API_KEY = os.getenv("SWIPIES_API_KEY")
API_URL = "https://api.swipies.ai"

# Upload a file to use as a knowledge source
def upload_file(file_path, description="Uploaded via Python API"):
    with open(file_path, 'rb') as file:
        files = {
            'file': (os.path.basename(file_path), file)
        }
        data = {
            'apiKey': API_KEY,
            'description': description
        }
        
        response = requests.post(
            f"{API_URL}/upload",
            files=files,
            data=data
        )
        
        if response.status_code == 200:
            return response.json()
        else:
            return f"Error: {response.status_code} - {response.text}"

# List all your uploaded files
def list_files():
    response = requests.get(
        f"{API_URL}/files",
        params={"apiKey": API_KEY}
    )
    
    if response.status_code == 200:
        return response.json()["files"]
    else:
        return f"Error: {response.status_code} - {response.text}"

# Example usage
file_path = "./data/product_documentation.pdf"
upload_result = upload_file(file_path)
print(f"Upload result: {upload_result}")

files = list_files()
print(f"Your files: {files}")
Batch Processing

Process multiple queries efficiently with batching:

import requests
import os
from dotenv import load_dotenv
import time
from concurrent.futures import ThreadPoolExecutor

load_dotenv()
API_KEY = os.getenv("SWIPIES_API_KEY")
API_URL = "https://api.swipies.ai"

# Function to process a single query
def process_query(query):
    response = requests.post(
        f"{API_URL}/chat",
        json={
            "message": query,
            "apiKey": API_KEY
        }
    )
    
    if response.status_code == 200:
        return {
            "query": query,
            "answer": response.json()["answer"]
        }
    else:
        return {
            "query": query,
            "error": f"Error: {response.status_code} - {response.text}"
        }

# Process multiple queries in parallel
def batch_process(queries, max_workers=5):
    results = []
    
    with ThreadPoolExecutor(max_workers=max_workers) as executor:
        futures = [executor.submit(process_query, query) for query in queries]
        for future in futures:
            results.append(future.result())
    
    return results

# Example usage
queries = [
    "What is the Swipies AI Platform?",
    "How can I integrate Swipies with my website?",
    "What programming languages are supported?",
    "What are the pricing plans?"
]

results = batch_process(queries)
for result in results:
    print(f"Q: {result['query']}\nA: {result.get('answer', result.get('error'))}\n")

Authentication

All API requests require authentication using an API key. You can include your API key in requests using one of the following methods:

Storage Management

The Swipies AI Platform provides efficient storage management for your data sources and AI agents. Understanding how storage works will help you optimize your usage and manage costs effectively.

Storage Limits

Each account has a storage allocation based on the subscription plan:

Plan Storage Limit File Size Limit Additional Storage
Free 1 MB 500 KB Not available
Standard 50 MB 5 MB $5 per 10 MB
Pro 250 MB 25 MB $3 per 10 MB
Enterprise 1 GB+ 100 MB Custom pricing
Storage Usage

Storage usage is calculated based on the raw size of uploaded files plus the processed embeddings generated from your data. Embeddings typically add approximately 15-20% to your storage usage.

Optimizing Storage Usage

Follow these best practices to optimize your storage usage:

  1. Compress files before uploading

    Compress large documents, especially those with images, before uploading to reduce storage consumption.

  2. Extract relevant content

    Instead of uploading entire documents, extract only the most relevant sections to create more focused knowledge sources.

  3. Use the Website Parser

    The website parser is optimized to extract only the relevant content from web pages, ignoring navigation elements, ads, and other non-essential components.

  4. Review and remove unused data

    Regularly review your data sources and remove any that are no longer needed or have been superseded by newer versions.

Storage Monitoring

You can monitor your storage usage in real-time from the Account section of your dashboard. The storage usage chart shows:

  • Total storage used vs. available
  • Storage breakdown by data type (files, websites, text, etc.)
  • Historical storage usage trends
Storage Optimization Tip

If you're approaching your storage limit, consider using the Text Summarization feature to condense large documents into smaller, more focused knowledge sources while preserving the essential information.
